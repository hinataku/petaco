# ペタコ 開発の経緯

## 概要
複数の定型文をストックし、それぞれに割り当てたショートカットキーで
どこでもワンクリック貼り付けできるMac用アプリ「ペタコ」。

## 決まった仕様（やり取りの流れ）

1. **きっかけ**
   既存のクリップボードマネージャー（Maccy, Clipy など）を検討したが、
   「定型文ごとに別々のショートカットキーを割り当てたい」という要望には
   既存アプリでは対応していなかったため、自作することに。

2. **基本仕様**
   - 通常のウィンドウアプリ（メニューバーのみの常駐ではなく、画面を持つ）
   - 定型文を複数登録できる
   - 各定型文に個別のショートカットキーを割り当てられる
   - デフォルトは ⌘1, ⌘2, ⌘3... と自動割り当て、後から好きなキーに変更可能

3. **名前**
   アプリ名は「ペタコ」に決定。

4. **履歴機能の追加**
   - アプリを開いたときに「登録一覧」と「貼り付け履歴」の両方を表示
   - 履歴はペタコ経由で貼り付けた定型文の履歴
   - 同じ内容を貼り付けたら、古い履歴は消えて最新の1件だけが残る

5. **永続化の確認 + 自動起動**
   - 登録一覧はもともとJSONファイルに保存されており、PC再起動後も保持される仕様
   - 新たに「PC起動時に自動的に立ち上げる」チェックボックスを追加
     （`SMAppService` でログイン項目に登録）

6. **メニューバー常駐化**
   - ウィンドウを閉じてもアプリが終了しないように変更
   - Dockには表示せず、メニューバーにアイコンとして常駐
   - 左クリックでウィンドウ表示、右クリックで「開く / 終了」メニュー

## 最終的なファイル構成
- `PetacoApp.swift` - アプリ本体のエントリポイント
- `AppDelegate.swift` - メニューバー常駐・Dock非表示・終了メニュー制御
- `ContentView.swift` - 登録一覧 + 履歴のメイン画面
- `Snippet.swift` - 定型文のデータモデル
- `SnippetStore.swift` - 定型文の保存・読み込み（JSON）
- `SnippetEditView.swift` - 定型文の追加・編集シート
- `KeyCaptureView.swift` - ショートカットキーをキャプチャするUI
- `KeyCodeMap.swift` - キーコードと文字の対応表
- `HotkeyManager.swift` - グローバルホットキーの登録・監視（Carbon API）
- `PasteManager.swift` - クリップボード経由でのペースト実行
- `PasteHistoryItem.swift` - 履歴1件のデータモデル
- `PasteHistoryStore.swift` - 履歴の保存・読み込み（重複排除あり）
- `LaunchAtLoginManager.swift` - ログイン時自動起動のON/OFF管理（SMAppService）

## 今後の検討候補（未実装）
やり取りの中で挙がったが、まだ手をつけていない案:
- 検索/フィルター機能
- 定型文の並び替え（ドラッグ&ドロップ）
- カテゴリ分け
- 定型文一覧の書き出し/読み込み（バックアップ・移行用）
- 複数行テンプレート + 変数差し込み（例: `{name}様、お世話になっております`）

## セットアップ方法
詳細手順は同梱の `README.md` を参照。
概要は「Xcodeで新規macOS Appプロジェクト(`Petaco`)を作成 → 同梱の
`.swift`ファイルを追加 → `Carbon.framework`を追加 → ビルド・実行」。
